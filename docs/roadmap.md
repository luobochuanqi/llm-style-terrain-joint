### 一、 训练过程（线下开发：建立数据库与纠错）

训练的本质是**“破坏数据，然后逼迫模型还原数据”**。在这个阶段，我们要同时喂给模型“题目（文本）”和“正确答案（高程+纹理）”。

**输入：** 真实三元组数据 `(Prompt字符串, 真实高度图矩阵, 真实纹理图矩阵)`
**目的：** 更新 8 通道 U-Net 和交叉注意力层的权重（Weight）。

**具体流水线（每个 Batch 的运转过程）：**

1.  **文本编码（题目翻译）：**
    * `Prompt` 进入双分支 CLIP。
    * 输出：`全局特征向量` 和 `细节特征向量`。暂时存放在显存中备用。
2.  **图像压缩（答案打包）：**
    * `真实纹理图 (3通道)` 进入 **SD VAE 编码器** $\rightarrow$ 压缩成 `纹理隐向量 (4x64x64)`。
    * `真实高度图 (1通道)` 进入 **你魔改的 VAE 编码器** $\rightarrow$ 压缩成 `高度隐向量 (4x64x64)`。
    * **【拼接】**：用 `torch.cat` 将两者按通道拼接，得到干净的 `联合隐向量 $z_0$ (8x64x64)`。
3.  **前向加噪（破坏答案）：**
    * 系统随机生成一个时间步 $t$（比如第 500 步）。
    * 系统生成一团纯随机的高斯噪声 $\epsilon$。
    * 把噪声 $\epsilon$ 按照时间步 $t$ 的强度，混入联合隐向量 $z_0$ 中，得到一张半毁坏的脏图 $z_t$。
4.  **模型预测（做题）：**
    * 把脏图 $z_t$、时间步 $t$、以及第 1 步翻译好的`文本特征向量`，一起喂给 **8 通道 U-Net**。
    * U-Net 经过复杂的计算，输出一个预测结果：**“我猜刚才加进去的噪声 $\epsilon_{pred}$ 是这样的。”**
5.  **计算误差与反向传播（对答案与自我修正）：**
    * 计算真实噪声 $\epsilon$ 和预测噪声 $\epsilon_{pred}$ 之间的误差（Loss）。
    * 触发反向传播（Backpropagation），优化器（AdamW）根据误差微调 U-Net 里的几亿个参数。
    * 循环往复几十万次，直到 U-Net 预测噪声极度精准。

### 二、 使用过程（线上推理：端到端生成）

这是模型部署上线后的状态。在这个阶段，模型**永远见不到**真实的图片，它完全依靠学到的参数常识来“凭空捏造”。

**输入：** 只有一段纯文本 `(Prompt字符串)`
**输出：** 生成的 `(512x512 高度图, 512x512 纹理图)`

**具体流水线（单次 API 调用过程）：**

1.  **下达指令：**
    * 用户输入 `Prompt`（例如：“广东丹霞地貌，红色平顶方山...”）。
    * 双分支 CLIP 将其翻译为向量指令，发给 U-Net 的各个注意力层。
2.  **准备初始石料（无中生有）：**
    * 系统在内存中，直接 `torch.randn` 初始化一个**纯随机的、形状为 8x64x64 的高斯噪声张量**，我们叫它 $z_T$。此时它就是一团杂乱无章的数字。
3.  **循环降噪（U-Net 雕刻过程）：**
    * 启动一个 50 步的循环（DDIM 采样器）。
    * **第 1 步**：U-Net 看着文本指令，观察纯噪声 $z_T$，预测出多余的噪声，并将其减去，得到稍微清晰一点的 $z_{T-1}$。
    * **第 2 到 49 步**：不断重复这个“观察 -> 预测 -> 减去噪声”的过程。在文本指令的引导下，U-Net 的 8 个通道会协同演化，确保第 1-4 通道（高程）和 5-8 通道（纹理）在物理位置上匹配。
    * **第 50 步**：最终得到一个清晰、无噪点的联合隐向量 $z_0$（8x64x64）。
4.  **拆分与解压（成品输出）：**
    * 将 8 通道的 $z_0$ 从中间劈开，恢复成两个 4 通道的隐向量。
    * 前 4 个通道 $\rightarrow$ **魔改 VAE 解码器** $\rightarrow$ 放大并反归一化，输出真实海拔数据（高度图）。
    * 后 4 个通道 $\rightarrow$ **SD VAE 解码器** $\rightarrow$ 放大并输出 RGB 图像（纹理图）。

### 三、 核心差异对比表

| 维度 | 训练过程 (Training) | 使用过程 (Inference) |
| :--- | :--- | :--- |
| **主要目标** | 计算 Loss 并更新 U-Net 权重 | 利用定型的权重，生成具体图片 |
| **数据流向** | 图像 $\rightarrow$ VAE 编码 $\rightarrow$ U-Net $\rightarrow$ Loss | 噪声 $\rightarrow$ U-Net 循环去噪 $\rightarrow$ VAE 解码 $\rightarrow$ 图像 |
| **VAE 的作用** | 仅使用 **Encoder（编码器）** 压缩图片 | 仅使用 **Decoder（解码器）** 解压隐向量 |
| **时间消耗** | 几周（跑几百万次迭代） | 几秒钟（循环几十步采样） |
| **内存/显存** | 极大（需保存所有神经元的梯度，用于反向传播） | 较小（不计算梯度，开启 `torch.no_grad()`） |
