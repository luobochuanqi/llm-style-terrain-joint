### Phase 1: 基础设施与环境基线
**目标**：配置好得心应手的现代开发环境，并跑通最基础的预训练模型。

- [ ] **安装地学与图像依赖**：安装 `rasterio`, `richdem`, `opencv-python` 等用于处理高程和纹理数据。
- [ ] **理解 Latent 维度**：在上述脚本中打断点或 `print`，观察并记录 VAE 压缩后的 `latent` 张量形状（通常是 `[1, 4, 64, 64]`），这是你后续魔改对齐的“参考系”。

### Phase 2: 数据流水线工程
**目标**：不碰模型，先把输入给模型的数据喂对。这是最容易踩坑的地方。

- [ ] **获取小样本测试集**：不要一上来就搞几万张，先手动去 Copernicus 和 Sentinel 下载 100 对喀斯特/丹霞地貌的高程图 (DEM) 和纹理图，裁剪为 512x512。
- [ ] **编写数据清洗脚本**：写一个 CLI 脚本，剔除无效值和云量超标的样本。
- [ ] **实现核心归一化逻辑**：**【关键】** 编写一个函数，将真实海拔值严格按照 $H_{max}=3000m$ 的公式映射到 `[-1, 1]`。将纹理图也归一化到 `[-1, 1]`。
- [ ] **构建 PyTorch Dataset**：编写一个继承自 `torch.utils.data.Dataset` 的类。它的 `__getitem__` 方法每次必须准确返回一个包含 `text`, `height_tensor`, `texture_tensor` 的字典。
- [ ] **DataLoader 可视化测试**：用 `DataLoader` 跑一个 Batch，写个逆归一化脚本把张量存成图片看一眼，确保送进模型前的数据没有变成全黑或全白的噪点。

### Phase 3: 高度图专用 VAE 的手搓与训练
**目标**：解决高程数据压缩失真的问题。

- [ ] **源码提取**：从 `diffusers` 库中把 `AutoencoderKL` 的相关代码剥离出来，或者直接继承它。
- [ ] **通道修改**：将 VAE 第一层卷积的 `in_channels` 从 3 改为 1，最后一层的 `out_channels` 从 3 改为 1。保证中间输出的隐向量维度依然是 `[4, 64, 64]`。
- [ ] **手写特殊 Loss**：在 `losses` 文件夹中实现地形几何约束损失 $\mathcal{L}_{geo}$。利用 PyTorch 张量操作计算相邻像素差值，模拟坡度约束。
- [ ] **闭环训练 VAE**：只用高度图数据（不需要文本和纹理），训练这个修改版 VAE。
- [ ] **质量验收**：输入一张含有陡崖的真实高度图，压缩再解压，对比重建结果。如果陡崖边缘清晰，这一关才算过。

### Phase 4: U-Net 魔改与阶段一预训练
**目标**：打通隐空间联合生成的链路，让模型学会在没有指令的情况下“随机捏地形”。

- [ ] **模型组装**：实例化标准 SD 1.5 U-Net。
- [ ] **输入层魔改**：将 U-Net `conv_in` 的输入通道从 4 改为 8。
- [ ] **输出层魔改**：将 `conv_out` 的输出通道从 4 改为 8。
- [ ] **权重初始化策略**：**【关键】** 编写代码，把 SD 预训练权重加载到前 4 个通道（对应纹理），后 4 个通道（对应高度图）进行零初始化。冻结前 4 个通道的梯度。
- [ ] **拼接隐向量**：在 `forward` 函数中实现 `torch.cat([latent_height, latent_texture], dim=1)`。
- [ ] **Stage 1 训练闭环**：屏蔽文本输入，将配对的高度图和纹理图送入双 VAE 压缩，拼装后喂给 8 通道 U-Net 计算降噪 MSE Loss。跑通第一个 Epoch 并观察 Loss 下降趋势。

### Phase 5: 细粒度文本注入与微调
**目标**：实现“言出法随”，让自然语言精准控制地形。

- [ ] **集成 OpenCLIP**：引入微调好的文本编码器，将 Prompt 转化为 768 维的特征向量。
- [ ] **解冻注意力层**：解冻 U-Net 中的 Cross-Attention 层参数。
- [ ] **实现双分支注入**：修改 U-Net 的注意力机制代码，让全局语义向量作用于所有层，细节语义向量通过线性层映射后仅注入第 3-6 层。
- [ ] **Stage 2 训练闭环**：接入【文本-高度-纹理】三元组数据，全链路微调。
- [ ] **联合一致性 Loss 开发**：加入皮尔逊相关系数的 $\mathcal{L}_{geo\_consist}$ 计算逻辑，进一步约束模型。

### Phase 6: 推理部署与封装
**目标**：把零散的脚本打包成优雅的、可以直接调用的 API 或 CLI 工具。

- [ ] **编写 CustomPipeline**：参考 `StableDiffusionPipeline`，重写一套自己的推理流程：接收字符串 -> CLIP 编码 -> 采样 8 通道高斯噪声 -> 循环 50 步 DDIM 去噪 -> 拆分 8 通道隐向量 -> 双 VAE 分别解码 -> 逆归一化。
- [ ] **后处理工程**：加入中值滤波代码，平滑高度图的异常噪点。
- [ ] **成果验收**：输入 `广东丹霞地貌，红色平顶方山...`，检查输出配对的 `.tiff` 和 `.png`。
